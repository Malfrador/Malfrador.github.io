Mythical Hunt on itch.io

Mythical Hunt is a co-op multiplayer RTT game developed in Unreal Engine over the course of 12 weeks. The game features a vampire/werewolf theme. Players can freely switch between four different characters, combining their abilities to defeat enemies and make it through the levels. The game, made in Unreal Engine 5.3, features four abilities per character, Multiplayer using Steamworks, and a variety of enemies.

My role

In our team of seven, I was the lead programmer, managing and teaching two other programmers. We mostly worked in Blueprints, although some more critical aspects of the game were done in C++ by me.
I mainly was responsible for the abilities, multiplayer and UI, as well as general systems.

Developing character abilities using GAS

Some example use of abilities in the debugging level.

One of my responsibilities was implementing all 16 abilities for the characters. For this I choose to use Unreal Engines Gameplay Ability System, as that provided good support for multiplayer networking out of the box. This was my first time using GAS, so it was a challenge at first.
The challenge was mainly due to the lack of documentation available for GAS, which meant that I had to read the GAS source code and rely on incomplete third party documentation.

For example, a completely undocumented caveat meant that the GAS lost its reference to the Ability System, the component attached to a character that is responsible for using abilities and tracking gameplay effects, when the controller of that character changed. This in turn lead to a crash of the game. Tracking this issue down proved to be quite difficult, as the crash happened in the Unreal Engine source code, and the call stack was not very helpful.
After a lot of debugging and reading the source code, I found the issue and was able to fix it by refreshing the reference whenever a character was switched.

In the end, I was able to implement all the 16 abilities in time, further iterating upon them by adding more features such as quick-cast mode based on initial testing feedback.

Co-op multiplayer using Steamworks

From the start, we wanted co-op Multiplayer in Mythical Hunt. As I was the only developer with any multiplayer experience, this responsibility naturally fell to me.
As such, I was responsible to ensure that all gameplay-relevant systems used RPCs in the correct places. We went with a "Listen Server" setup, where one of the clients also acted the server. This seemed suitable for our small-scale multiplayer with a maximum of four players. The listen server acts authoritative, meaning that it has the final especially regarding abilities. While not really necessary in terms of game-integrity for a game like Mythical Hunt, it reduced the possibilities of desync occurding. More for fun I also decided to make movement server-authorative, meaning that the client requests a position from the server, which the server confirms. To reduce visual lag, the client will also start moving at the same time. However, the server will correct the client if it tries to move in a different location than the server decided it should. This was not necessary for the game, but I wanted to try it out and it was a great learning experience.

A little demonstration of multiplayer working and correctly replicating the game.

To make multiplayer easily accessible, we decided to use Steamworks for the networking. This was a good choice, as it provided a lot of the functionality we needed out of the box. Implementing multiplayer was surprisingly issue-free. The most challenging part proved replicating abilities and their animations correctly, as the Unreal Engine implementation of this is not very well documented.
